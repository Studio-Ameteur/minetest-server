# Документация по эксплуатации Server Manager

Руководство по установке, запуску и использованию инструмента управления сервером Minetest.

---

## Установка

### Вариант 1 — Готовый exe (рекомендуется)

1. Перейдите в репозиторий на GitHub
2. Откройте вкладку **Actions** → последний успешный билд
3. Скачайте артефакт **MinetestManager**
4. Распакуйте и запустите `MinetestManager.exe`

Дополнительная установка не требуется.

### Вариант 2 — Из исходников

```bash
# Установить Python 3.11+
# https://www.python.org/downloads/

# Перейти в папку
cd tools/server-manager

# Установить зависимости
pip install -r requirements.txt

# Запустить
python main.py
```

---

## Окно входа

При запуске открывается окно подключения к серверу.

| Поле | Описание |
|------|----------|
| IP-адрес сервера | Публичный IP вашего сервера |
| Логин | Пользователь SSH (обычно `root`) |
| Пароль | Пароль SSH |

После заполнения нажмите **Подключиться**. Программа проверит данные напрямую через SSH — если данные неверные, появится сообщение об ошибке.

> Данные хранятся только в памяти программы и не сохраняются на диск.

---

## Главное окно

### Статус сервера

В верхней части отображается текущий статус сервиса Minetest:

| Статус | Значение |
|--------|----------|
| 🟢 Работает | Сервер запущен и принимает игроков |
| 🔴 Остановлен | Сервер выключен |
| 🔴 Ошибка | Сервис упал с ошибкой |
| ⚪ Неизвестно | Не удалось получить статус |

Статус обновляется автоматически каждые 10 секунд.

### Кнопки управления

| Кнопка | Действие |
|--------|----------|
| ▶ Запустить | Запускает сервис Minetest |
| ■ Остановить | Останавливает сервис |
| ↺ Перезагрузить | Перезапускает сервис |
| 📊 Статистика | Открывает окно статистики |

### Нагрузочное тестирование

Позволяет имитировать максимальную нагрузку на сервер для проверки производительности.

1. Укажите длительность теста в секундах (10–300)
2. Нажмите **⚡ Имитировать нагрузку**
3. Подтвердите запуск в диалоговом окне
4. Следите за метриками в окне статистики

> ⚠️ Во время стресс-теста сервер работает под максимальной нагрузкой. Не запускайте тест в часы пик.

### Лог действий

В нижней части главного окна отображается история всех выполненных команд с временными метками.

## Окно статистики

Открывается кнопкой **📊 Статистика**. Показывает текущие метрики сервера.

| Метрика | Описание |
|---------|----------|
| CPU | Загрузка процессора в % |
| RAM | Использование памяти (занято/всего МБ) |
| Диск | Использование диска (занято/всего) |
| Uptime | Время работы сервера без перезагрузки |
| Игроков за час | Количество подключений за последний час |
| Нагрузка 1 мин | Load average за 1 минуту |
| Нагрузка 5 мин | Load average за 5 минут |
| Нагрузка 15 мин | Load average за 15 минут |

Нажмите **↺ Обновить** для получения актуальных данных.

**Ориентиры по нагрузке (load average):**
- Значение < количества ядер — норма
- Значение = количеству ядер — сервер загружен
- Значение > количества ядер — перегрузка, нужно масштабирование

## Сборка exe из исходников

Если нужно собрать exe вручную:

```bash
cd tools/server-manager
pip install -r requirements.txt
pyinstaller --onefile --windowed --name "MinetestManager" main.py
```

Готовый файл появится в папке `dist/MinetestManager.exe`.

## Частые проблемы

**Не подключается к серверу**
- Проверьте правильность IP-адреса
- Убедитесь что порт 22 (SSH) открыт на сервере
- Проверьте логин и пароль

**Кнопки управления не работают**
- Убедитесь что пользователь имеет права на управление systemd
- Проверьте что сервис называется `minetest` (можно изменить в `server.py`)

**Статистика показывает прочерки**
- Проверьте подключение к серверу
- Нажмите кнопку обновления в окне статистики

**Стресс-тест не запускается**
- На сервере должен быть установлен пакет `stress`
- Программа устанавливает его автоматически, но требуются права root
